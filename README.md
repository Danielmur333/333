# Система обращений пользователей

Веб-приложение для управления обращениями пользователей с разделением ролей "Пользователь" и "Администратор".

## 🚀 Функционал сайта

- **Страница входа** с кнопкой для перехода к авторизации
- **Форма авторизации** (логин/пароль) с валидацией
- **Личный кабинет пользователя**:
  - Отображение аватарки и имени пользователя
  - Кнопка для создания нового обращения
  - Сохранение обращений в браузере
- **Панель администратора**:
  - Просмотр всех обращений от пользователей
  - Сортировка обращений по дате (новые/старые сверху)
  - Отображение информации об авторе обращения
- **Выход из аккаунта** с очисткой сессии

## 🛠 Стек технологий

| Технология | Назначение | Почему выбран |
|------------|------------|----------------|
| **HTML5** | Структура страниц | Универсальный стандарт для вёрстки веб-интерфейсов |
| **CSS3** | Стилизация и адаптивный дизайн | Позволяет создать современный, приятный интерфейс без дополнительных зависимостей |
| **JavaScript (Vanilla ES6)** | Вся логика приложения | Выбран осознанно, чтобы:<br>• Не усложнять проект сборщиками и транспиляторами<br>• Продемонстрировать понимание нативного JS<br>• Обеспечить максимальную производительность<br>• Упростить развёртывание на GitHub Pages |
| **LocalStorage** | Хранение данных | • Не требует бэкенда и сервера<br>• Данные сохраняются между сессиями<br>• Идеально для демонстрации прототипа |
| **SessionStorage** | Хранение сессии пользователя | Безопасное временное хранение данных о текущем пользователе |
| **GitHub Pages** | Хостинг и деплой | • Бесплатный хостинг статических сайтов<br>• Автоматическая публикация из репозитория<br>• HTTPS из коробки |

## 📚 Библиотеки

**Проект выполнен без использования внешних библиотек и фреймворков** — только чистый HTML/CSS/JavaScript.

Это сознательное решение, позволяющее:
- Продемонстрировать навыки работы с нативным DOM API
- Избежать лишних зависимостей и "чёрных ящиков"
- Обеспечить максимальную лёгкость и скорость загрузки
- Упростить проверку кода преподавателем

## 🧪 Тестовые данные

Для проверки функционала используются предустановленные учётные записи:

| Роль | Логин | Пароль |
|------|-------|--------|
| Пользователь (User) | `user` | `user123` |
| Администратор (Admin) | `admin` | `admin123` |

## 🏗 Архитектура приложения

Приложение построено по принципу **SPA (Single Page Application)** с условной навигацией:

1. **State-менеджмент** — данные пользователя хранятся в `sessionStorage`, обращения — в `localStorage`
2. **Рендеринг** — динамическая отрисовка страниц через манипуляции с DOM
3. **Маршрутизация** — условная, на основе роли пользователя (user/admin)
4. **Компонентный подход** — каждая "страница" вынесена в отдельную функцию
